The agent loop — three phases, one trip

Loop agent — tiga fase, satu trip

智能体循环 — 一次行程三个阶段

This isn't a chatbot answering questions. It's an autonomous loop with tools, schedules, and decisions that fire whether or not the fisher is looking at his phone.

Ini bukan chatbot tanya-jawab. Ini loop otonom dengan tools, jadwal, dan keputusan yang jalan walaupun nelayan gak buka HP.

这不是问答聊天机器人，而是带工具、调度和决策的自主循环 — 即使渔民不看手机也持续运转。

02 · At sea

Watch the boat

Awasi kapal

海上守护

Scheduled SMS heartbeats home with GPS pin. Weather change alerts on satellite SMS gateway. Mid-trip catch logging via voice — no keyboard, wet hands, sun glare.

Heartbeat SMS terjadwal ke keluarga + pin GPS. Alert perubahan cuaca via gateway SMS satelit. Log tangkapan tengah-trip via suara — tanpa keyboard, tangan basah, silau matahari.

定时短信心跳与 GPS 位置发送给家人。卫星短信网关推送天气突变预警。语音记录捕捞 — 无需键盘，湿手日晒皆可。

Cost-tier routing — built for cheap inference

Routing biaya — dirancang untuk inferensi murah

成本路由 — 为低成本推理而设计

Most of what this agent does is lookup, format, and schedule — work that belongs on a small, fast, cheap model. Heavy reasoning fires only when the trip needs it.

Mayoritas pekerjaan agent ini adalah lookup, format, jadwal — kerjaan untuk model kecil, cepat, murah. Reasoning berat cuma trigger kalau trip butuh.

智能体大部分工作是查询、格式化和调度 — 适合小而快、低成本的模型。仅在行程需要时才触发重推理。

Honest framing

  • What exists today: this landing, an interactive scripted demo of the 3-phase loop, a public BMKG API probe, and a costed routing plan. No production users yet.
  • Yang ada hari ini: landing ini, demo interaktif 3-fase yang ter-skrip, probe API BMKG public, dan rencana routing cost. Belum ada user produksi.
  • 当前已有：本落地页、3 阶段交互脚本演示、BMKG 公共 API 探测、成本路由计划。尚无生产用户。
  • What's intentionally not in scope: sonar integration (smallholder boats don't carry it), sat-phone hardware (we lean on cheap GSM SMS gateways), and any kind of marine safety authority claim.
  • Yang sengaja gak masuk scope: integrasi sonar (kapal kecil gak punya), hardware sat-phone (kami pakai gateway SMS GSM murah), dan claim jadi otoritas keselamatan laut.
  • 故意不在范围内：声纳集成（小船没有）、卫星电话硬件（采用廉价 GSM 短信网关）、任何形式的海事安全权威声明。
  • The biggest open risk: connectivity at sea is genuinely unreliable. Our design treats it as a constraint (offline-first cache + SMS fallback) rather than a problem we'll "solve later".
  • Risiko terbuka terbesar: konektivitas di laut beneran goyang. Desain kami treat ini sebagai constraint (cache offline-first + fallback SMS), bukan problem yang "diberesin nanti".
  • 最大开放风险：海上连接确实不稳定。我们的设计把它当作约束（离线优先缓存 + 短信兜底），而非"以后再解决"的问题。
